# 日報作成 Skill

## 手順

1. **対象日付の決定**:
   - `$ARGUMENTS` に YYYY-MM-DD があればそれを使う
   - なければ今日の日付（システム日付）

2. **既存ファイルチェック**:
   - `daily_reports/<YYYYMMDD>.md` が**すでに存在すれば**、そのまま開いてユーザーに「既に今日の日報があります」と報告して停止
   - 過去の日報は**絶対に編集しない**（CLAUDE.md ルール）

3. **前日レポートの読み込み**:
   - `daily_reports/[0-9]*.md` にマッチする最新ファイルを取得（日報以外の .md を誤って拾わないため glob を限定する）
   - 「次にやること」「未完了 TODO」「数値まとめ」「戦略・ロードマップ」を抽出

4. **直近の実験結果を引き継ぐ**（並列ロード可能、1M ctx を活かす）:
   - `workspace/exp*/SESSION_NOTES.md` のうち、前日以降に更新があったものを読む
   - `submit/SUBMISSIONS.md` の最新行
   - `claudeSummary.md` のベストスコア履歴

5. **今日の日報を生成** （CLAUDE.md のテンプレートに準拠）:

```markdown
# 日報 YYYY-MM-DD

## コンペ情報
- **コンペ**: <KAGGLE_DIRECTION.md から取得>
- **締切**: YYYY-MM-DD（残り N 日）
- **評価指標**: <KAGGLE_DIRECTION.md から取得>
- **LB状況**: <前日の LB 状況を引き継ぎ>

## 今日やったこと
<!-- セッション開始時は空。作業しながら追記する -->

## 数値まとめ
| 実験 | データ量 | CV | LB | 状態 |
|------|---------|-----|-----|------|
<!-- 前日のテーブルから「進行中」「直近」のみ引き継ぎ。完了済みで決着したものは省略可 -->

## 戦略・ロードマップ
<!-- 前日から引き継ぎ。アップデートがあれば書き換える -->

## 判断・知見
<!-- 今日得られた重要な判断や知見。空で開始 -->

## データ在庫
<!-- 前日から引き継ぎ -->

## 次にやること
<!-- 前日の「次にやること」のうち未完了のものを引き継ぎ -->
- [ ] ...
```

6. **報告**:
   - 生成したファイルパス
   - 引き継いだ未完了 TODO 数
   - 直近の実験結果サマリ（CV/LB の動向 1-2 行）

## 注意

- 既存の日報を**絶対に上書きしない**
- 前日が複数日前（土日跨ぎ等）でも、最新の日報を引き継ぐ
- 「今日やったこと」は空で開始する。Claude や人間が作業中に追記する想定
- 過去の日報は履歴。修正したい場合は「日付を訂正」ではなく**別日付で訂正版を作る**
