# コンペ概要 — The Pokémon Company PTCG AI Battle Challenge

最終更新: 2026-06-21

## 1. プラットフォーム

- **Kaggle**（2トラック並行）
  - Simulation トラック: https://www.kaggle.com/competitions/pokemon-tcg-ai-battle
  - Strategy トラック: https://www.kaggle.com/competitions/pokemon-tcg-ai-battle-challenge-strategy
- Simulation はラダー対戦でスコアリング。Strategy は戦略レポートを提出し上位8チームがファイナルへ進出。

## 2. タスク定義

- **タスク**: ポケモンカードゲーム（TCG）を対戦プレイするAIエージェントを構築する
- **入力**: ゲーム状態を表す観測辞書 `obs_dict`（詳細は § 8 参照）
- **出力**: `list[int]` — 合法手の中から選んだ選択肢インデックスのリスト
- **ゲームルール**: 標準フォーマット（60枚デッキ、主催者提供カードプールのみ合法）
- **評価単位**: 1エピソード = 1ゲーム（1試合 = ベストオブN か 1本勝負かは要確認）
- **プレイヤー数**: 2人対戦（1対1）
- **1プレイヤーあたりの時間制限**: 1試合あたり最大10分

## 3. データ所在

- **提供物**: 主催者提供カードプール（使用可能カード一覧）、ゲームエンジン `cg/`、SDK（シミュレータ）
- **エンジン**: cabt Engine（Pokemon TCG battle simulator for kaggle-environments）
- **環境パッケージ**: `kaggle-environments==1.30.1`
- **カードプール**: 主催者が指定したリストのカードのみ合法（標準フォーマット相当）
- **データフォーマット**: カードリスト CSV
- **ローカル配置先**: `datasets/`（提供 SDK をここに展開する）
- **ライセンス**: 推測: 競技利用のみ（要確認）

## 4. 評価指標

- **名称**: スキルレーティング（Elo 類似、ガウシアン型）
- **定義**: 各エージェントのスキルを $N(\mu, \sigma^2)$ でモデル化
  - 初期値: $\mu_0 = 600$、$\sigma$ は試合を重ねるごとに縮小
  - 勝ち → $\mu$ 上昇、負け → $\mu$ 下降、引き分け → 中間調整
- **マッチング**: 近いスキルレーティングのエージェント同士がマッチング（ラダー方式）
- **最終スコア**: $\mu$ の値がリーダーボードスコア
- **ローカル代理指標**: 固定相手プール（`reference_sim/opponents/`）に対する勝率

## 5. 提出形式

- **種別**: (D) Simulation エージェント
- **提出ファイル**: `submission.tar.gz`
- **内部構成**:
  ```
  submission.tar.gz
  ├── main.py      # エージェント本体（agent(obs_dict) -> list[int] を実装）
  ├── deck.csv     # 使用するデッキ（カード60枚のIDリスト）
  └── cg/          # 主催者提供のゲームエンジンライブラリ
  ```
- **作成コマンド**: `tar -czvf submission.tar.gz main.py deck.csv cg/`
- **提出上限**: 1日あたり5回、うち最新2件のみが評価対象
- **必須条件**: main.py はクラッシュしないこと（合法フォールバックを必ず返す）。エージェント内で stdout に print しない（採点ログを汚す）

## 8. Simulation 追加情報

- **エンジン / 環境**: cabt Engine（kaggle-environments ベース、`pokemon-tcg-ai-battle`）
  - `kaggle-environments==1.30.1` が必要
  - ローカルでは提供 SDK で同一ロジックを使って訓練・テスト可能
- **観測空間** (`obs_dict`): 詳細構造は要確認（Kaggle 公式 SDK を参照のこと）
  - ゲーム状態（場のポケモン、手札、エネルギー、残りHP など）を含むと推測
  - **エンジンが毎ターン合法手を提示**する設計（不正手を選ぶ必要なし）
- **行動空間**: `list[int]` — エンジンが提示した合法手インデックスのリスト
  - 非合法手のペナルティ: エンジンが合法手のみ提示するため原則不要（ただしクラッシュはペナルティ）
- **エピソード / ターン構造**: 交互手番、60枚デッキを使ったTCG標準フォーマット
- **レーティング方式**: ガウシアン型スキルレーティング（$\mu_0=600$）、ラダーマッチング
  - 1日あたりマッチ数: 自動（提出後継続的に対戦）
- **エージェントI/F**:
  ```python
  def agent(obs_dict: dict) -> list[int]:
      # obs_dict: ゲーム状態（エンジン提供）
      # return: 選択した合法手インデックスのリスト
      ...
  ```
  - 1手あたりのタイムアウト: 不明（10分/試合の枠内）
  - メモリ制限: 要確認
  - エージェントファイルサイズ上限: 要確認
- **ローカル評価の代理指標**: 固定相手プール（`reference_sim/opponents/`）に対する勝率で代替
