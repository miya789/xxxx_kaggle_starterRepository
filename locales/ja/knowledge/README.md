# Knowledge Wiki（ストック層）

このディレクトリは **蒸留された再利用可能な知識** を貯める「ストック層」。
1トピック1ファイルの原子ページで構成し、`INDEX.md` を検索用の目次として使う。
`memory/`（Claude Code のセッション横断メモ）と同じ型を、**コンペ固有の知識**に展開したもの。

## フロー と ストック を分ける

| 層 | 置き場 | 性質 | いつ書く |
|----|--------|------|---------|
| **フロー** | `daily_reports/YYYYMMDD.md` | 時系列ログ。追記専用、編集しない | その日に起きたこと・数値を即記録 |
| **ストック** | `knowledge/**.md` | 蒸留知識。原子・重複排除・最新化 | フローから「再利用できる知見」を昇格 |

**原則: まず daily_report に書く → 蒸留して knowledge へ昇格する。**
daily_report は「何が起きたか（履歴）」、knowledge は「今わかっている事（再利用可能な結論）」。

## ディレクトリ

```
knowledge/
  INDEX.md          # 検索用の目次（1行/ページ + フック）。中身を読む前にまずここを見る
  README.md         # この説明
  _template.md      # 新規ページの雛形
  technique/        # 効いた/効かなかった手法（CV/LB差分つき）
  data/             # データの性質・リーク・fold設計の知見
  error/            # エラーパターンとその対処
  decision/         # 方針判断とその理由（なぜそれを選んだ/捨てたか）
```

カテゴリは増やしてよい（例: `metric/`, `pipeline/`）。増やしたら INDEX に節を足す。

## ページの作法

- **原子的**: 1ページ＝1トピック。大きくなったら分割する
- **frontmatter 必須**（`_template.md` 参照）: `id / title / category / status / tags / source / links / updated`
- **本文は短く蒸留**: 要点 → 根拠（数値） → 使い方 → 注意。だらだら書かない
- **相互リンク**: 関連ページは `[[id]]` で繋ぐ。存在しない id へのリンクも可（後で書く印）
- **出典を残す**: `source:` に daily_report の日付 / exp フォルダ / survey の論文を書く（追跡可能に）

## ライフサイクル（status）

`idea`（仮説） → `testing`（検証中） → `validated`（効果確認） / `rejected`（棄却） / `stale`（陳腐化）

棄却された知見も**消さずに `rejected` で残す**（同じ轍を踏まないため）。

## 検索の仕方（LLM向け）

1. まず `INDEX.md` だけ読む（全ページは読まない）
2. フックを見て関連ページを2-3個選ぶ
3. 選んだページだけロードする

これで知識が膨大になっても context を食わずに引ける。

## 蓄積・整理は `/wiki`

- `/wiki` … INDEX を見せて、足りていない知識の穴を提案
- `/wiki add <トピック>` … 知見を該当ページに蒸留（新規作成 or 更新）+ INDEX 更新
- `/wiki find <クエリ>` … 関連ページを検索してロード
- `/wiki promote` … 最新 daily_report から未昇格の知見を拾ってページ化を提案
- `/wiki consolidate` … 重複・矛盾・陳腐化を統合し INDEX を再生成（週1目安）
